How it works

Clone the repo.
Run /start. Pick up where you left off.

FonteOS is a vault structure and a set of behavioral rules — written in plain markdown. Claude loads it automatically every session.

01

Clone the repo

It's markdown files and a CLAUDE.md. No install, no dependencies, no configuration wizard. Open the folder as an Obsidian vault or any markdown editor.

git clone → open → done
02

Run /start

Claude reads your world: mission, active projects, last session state. It surfaces what's relevant, scopes to one project, and asks what you're working on. First run walks you through setup. Every run after is instant.

/start → context loaded → work begins
03

Work → /end → next session picks up cold

At the end of each session, /end saves your state: decisions made, work done, what's next. The next session reads that state and starts where you left off. No recap needed.

/end → state saved → cold start eliminated
Philosophy

Not a new system to learn.
A structure that grows with you.

FonteOS is unopinionated by design. No methodology to adopt, no process to follow. Just a clear structure that separates thinking from doing.

Two layers, one system

Everything splits into strategy (vision, priorities, knowledge) and execution (tasks, issues, status). The vault holds strategy. Your task manager holds execution. They stay in sync through /start and /end — no double entry, no drift.

Vision → Project → Task

Your north star lives in core.md. Projects (springs) break it into workable domains. Tasks live in your task manager. Claude navigates this hierarchy automatically — loading only what's relevant to the current project, not everything you've ever written.

Start small, add as you go

Day one: a core.md and one spring. That's it. Add streams when a project gets complex. Add intelligence files when research recurs. Add agents when you need specialization. The system earns its complexity — you never have to design it upfront.

Easy to maintain

Everything is plain markdown. No databases, no config files, no lock-in. If you can read a file, you can understand the system. If you can edit a file, you can change it. Monthly /audit catches orphans and drift. Weekly maintenance takes minutes.

Built for ADHD brains

Guardrails that keep you
shipping, not spiraling.

Solo founders with ADHD know the pattern: you start a task, discover an interesting adjacent problem, and two hours later you've built infrastructure for something you haven't shipped yet. FonteOS encodes the counter-patterns directly into how Claude operates.

Output typing

Ships vs. Builds vs. Learns

Every task gets labeled by output type. "Ships" go in front of a real human. "Builds" are internal infrastructure. "Learns" is research. The system tracks your ratio — and nudges you when you're building too much and shipping too little.

  • Weekly ship commitment enforced in daily cycle
  • Monday: commit to what ships this week
  • Friday: check if it actually shipped
Always-on

Pattern monitoring

Five failure patterns are tracked every session, not just when you run an audit. Claude flags them in real time and intervenes before they become a problem.

  • Scope expansion — "just one more thing" syndrome
  • Prerequisite chains — "I need X before I can do Y"
  • Abstraction escalation — solving the general case instead of the specific one
  • Build-without-shipping — infrastructure that never reaches a user
  • Conditional commitment — "I'll do X once Y is perfect"
Session protocol

Session deliverables

Every work session starts by declaring four things. This bypasses the executive function wall — no decision fatigue about what to do, just do the thing.

  • Specific deliverable — not "work on project" but "draft the hero section"
  • Type — ships or builds? Sets the accountability frame
  • Done definition — how you'll know it's finished
  • First 15-minute action — smallest possible on-ramp to start
Drift protection

Nudges and interventions

When the system detects drift, it doesn't just log it — it acts. Calibrated responses based on severity, from gentle reminders to hard stops.

  • Spring isolation — one project per session, cross-edits require confirmation
  • Context budget — ~5k tokens max, prevents over-loading
  • Correction loops — if Claude repeats a failed approach twice, it suggests a reset
  • /end comparison — declared deliverable vs. actual output, every session
Commands & skills

Your workflow, encoded
into repeatable operations.

Slash commands handle the recurring patterns. Skills handle the specialized tasks. Both are just markdown files you can read, modify, or extend.

Session commands
/start Load context, read session state, pull task manager issues, scope to a project, suggest work.
/end Save session state — decisions, work done, blockers, next priorities. Compare deliverable vs. actual output.
/cycle Automated daily scan. Classify issues, execute approved work, write a daily report. Runs on a schedule or manually.
/audit Monthly vault health check. Broken links, orphaned notes, stale references, CLAUDE.md drift. Fixes what it can, flags the rest.
/patterns Run a pattern-breaking self-audit. Check ships ratio, scope creep, prerequisite chains. Manual trigger for the always-on system.
/review Two-stage code and content review. Fast gap analysis, then deep counter-review. Catches what a single pass misses.
Skills & modes
/perplexity Deep research mode via Perplexity. Reddit mining, market research, competitive analysis. Cost-gated — asks before calling.
/youtube Analyze YouTube videos via Gemini. Extract key points, summarize, pull quotes. For research without watching the full video.
/story-edit Craft, review, or restructure stories, scripts, and founder content. Voice-aware — loads your writing profile automatically.
/humanize Rewrite AI-generated text to sound like a real human wrote it. Detects and removes 25+ AI writing patterns.
/note Create a new note in the vault with proper frontmatter, wikilinks, and placement. Quick capture without breaking flow.
/interview Structured interview mode for ambiguous features. Asks the right questions before planning, so you build the right thing.
Commands and skills are markdown files in .claude/commands/ and .claude/skills/. Read them, modify them, or write your own. The system is designed to be extended, not consumed.

Core benefits

What makes it work.

The fundamentals underneath everything else.

Persistent memory

Session state saved at /end, restored at /start. Decisions, work done, blockers, next priorities — all carried forward. No more "let me catch you up."

Focused context loading

A six-layer hierarchy tells Claude exactly what to load. ~5k tokens per session, not your entire vault. 90% of the window stays free for actual work.

Secure by default

No npm. No dependencies. No build step. No server. No API keys to manage. It's markdown files in a folder. Nothing to audit, nothing to patch.

Behavioral guardrails

Three-tier permission system: autonomous, propose-and-wait, never-allowed. Claude reads freely, proposes edits to important files, can't delete anything.

Specialized agents

Define AI agents in markdown — model, role, tools, context. Researcher, content writer, SEO strategist, operations. Your workforce, scoped and reusable.

Automated instructions

CLAUDE.md behavioral rules run every session. Pattern monitoring, permission tiers, scope limits — all encoded. Claude follows your system, not its defaults.

What it is not

No hype. No dependencies.

FonteOS is a working context for Claude Code — a structured environment, not a conversation interface. It assumes you're already in the terminal.
